Architecture Classification

System Intelligence & Design Principles

Handoverly is classified as a Clinical Decision Support System (CDSS) — augmenting clinical judgment through AI assistance while keeping the registered nurse fully accountable for every published outcome.

🧠
Intelligent Decision Support
AI-assisted layer that structures and refines clinical documentation. Dynamically adapts to resident profile, care requirements, and current workflow stage — not a passive static form.
✔ Confirmed
🔄
Interactive Workflow Engine
Multi-step progressive pipeline: record → generate → edit → approve → publish. Each stage is explicit, sequential, and human-controlled. No stage can be skipped or automated past.
✔ Confirmed
⚡
Fault-Tolerant AI Orchestration
5-provider cascade with graceful degradation. Failure of any single provider triggers immediate fallback to the next — ensuring continuous care documentation under all network conditions.
✔ Confirmed
👤
Human-in-the-Loop
All AI outputs require explicit RN review and approval before database persistence. No content is published without a deliberate human decision at the mandatory approval gate.
✔ Confirmed
🧩
Role-Driven Adaptive Interface
Interface adapts available controls and information density based on authenticated role (Admin/RN/Carer) and current workflow stage — presenting only contextually relevant actions.
✔ Confirmed
🚫
Autonomous / Self-Aware
The system does not make autonomous clinical decisions, learn from behaviour patterns, or act without human instruction. Full clinical accountability remains with the registered nurse at all times.
✘ Not Applicable

  • Full system access

  • Voice / text note recording
  • AI handover generation

  • Quick-register residents

End-to-End

Clinical Handover Workflow

The complete lifecycle from login to published handover, with the mandatory human decision at each stage.

🔐
Stage 01
Authentication
Staff selects facility → enters name + PIN. bcrypt compare server-side. Role determines redirect: Admin → /admin, RN → /shift, Carer → /dashboard. Session persisted in context + sessionStorage.
📋
Stage 02
Shift Registry — Resident Selection
RN views all active residents with today's status indicators. Filters by wing, name/room text, and status tab. Selects a resident to begin or update their handover record for the current shift.
🎙️
Stage 03
Note Recording — Voice or Text
RN records observations as task nodes: voice dictation (Web Speech API, en-AU, continuous mode) or manual text. Each node auto-refined by AI after dictation stops. Multiple nodes per session.
🤖
Stage 04
AI Processing — Handover Generation
Raw notes sent to /api/generate-handover. Provider cascade: Claude → OpenRouter → Groq → Ollama → Mock. Returns structured ISBAR + carer tasks + risk flags + urgency + follow-up questions.
✏️
Stage 05 — Human-in-the-Loop Gate
Review & Edit
RN reviews all AI output. Can edit any ISBAR section, modify/add/delete carer tasks, toggle category tags, override urgency level, and play tasks via TTS. This stage is mandatory — cannot be bypassed.
✅
Stage 06
Approve & Publish
RN approves. Shift type auto-detected from clock. Full record persisted: raw input, original AI output, RN-edited output, risk flags, urgency, submitter ID, approval timestamp, shift date.
📺
Stage 07
Live Dashboard — Real-Time Broadcast
Supabase Realtime instantly pushes the new handover to all connected clients. Carers see plain task lists. RNs see full ISBAR summaries + risk flags. Auto-sorted by urgency, colour-coded for priority.
